# Plain-Text Friendly Markdown Guide

## 1. Structure
# Level 1 Header (Main Topic)
## Level 2 Header (Sub-section)

## 2. Lists
- Use hyphens for simple bullets
- They are easy to read in raw text
- No special rendering required

1. Use standard numbers for steps
2. Most editors auto-increment these
3. Very clear in plain text

## 3. Emphasis
Use **bold** for strong points.
Use *italics* for subtle emphasis.

## 4. Separation
---
(The line above creates a visual break)

## 5. Technical Notes
Use `backticks` for technical terms or keyboard shortcuts.

> Use blockquotes for highlighting specific quotes or 
> external information you want to reference later.
