# Plain-Text Markdown Cheat Sheet

This guide highlights Markdown syntax that remains legible in raw text editors while rendering perfectly on a GitHub Pages site.

---

## 1. Headers
# Level 1 Header
## Level 2 Header
### Level 3 Header
This is what the markup looks like.

**Markup:**
# Level 1 Header
## Level 2 Header
### Level 3 Header

---

## 2. Bullet Lists
- Use a hyphen followed by a space
- It creates a clean, indented list
- Perfect for quick thoughts
This is what the markup looks like.

**Markup:**
- Item one
- Item two
- Item three

---

## 3. Numbered Lists
1. First step
2. Second step
3. Third step
This is what the markup looks like.

**Markup:**
1. First step
2. Second step
3. Third step

---

## 4. Bold and Italic
This is **bold text** and this is *italic text*.
This is what the markup looks like.

**Markup:**
This is **bold text** and this is *italic text*.

---

## 5. Horizontal Rules
The line below separates content blocks:
---
This is what the markup looks like.

**Markup:**
---

---

## 6. Blockquotes
> This is used for quotes or highlighting specific notes.
> It keeps the text wrapped together visually.
This is what the markup looks like.

**Markup:**
> This is a blockquote.

---

## 7. Inline Code
Use `backticks` to highlight technical terms or file names.
This is what the markup looks like.

**Markup:**
Use `backticks` for code spans.
